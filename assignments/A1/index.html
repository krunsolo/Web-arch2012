Web Standards

There are various reasons as to why the use of web standards are good for different stakeholders. Let us take a look at a few:

SEO

One of the primary reasons is SEO Correct and semantic use of(X)HTML. By using CSS to style your document, you are able to separate structure from presentation. A machine, in this case the search engine, can match semantic html tags such as <title> to return more effective search results. This way the search engine does not have to "read" through the entire content to find keywords, it can simply return the parts of the document that best describe it which is usually the title.

Accessibility

As Tim Berners-Lee has said, "the power of the Web is in its universality. Access by everyone regardless of disability is an essential aspect." Meaningful(X)HTML structure makes for a good start in accessibility. This can be done by use of CSS classes/ids and the new semantic HTML5 elements.

Stability

Web developers have to worry about forward and backward compatibility. Web standards offer a set of rules that every Web developer can follow, understand, and become familiar with: when one developer designs a site to the standards, another will be able to pick up where the former left off.

An important standard is the use of alt attributes for images, this is very important for accessibility reasons. It primarily affects disabled users. The alt attribute should provide a description of the respective image so that when the image is broken, not viewable by the user or a textual browser is used instead of a graphical browser; there is still some information retained about the image.

HTML Short History Lesson | source
HTML 3 (late 1995) was an ambitious effort on the part of Dave Raggett to upgrade the features and utility of HTML. However, it was never completed or implemented, although many features were integrated in the next "official" version of HTML, known as HTML 3.2.
HTML 3.2 was the next official version, integrating support for TABLES, image, heading and other element ALIGN attributes, and a few other finicky details. HTML 3.2 is the current "universal" dialect -- essentially all browsers understand HTML 3.2. IT was, however, missing some of the Netscape/Microsoft extensions, such as FRAMEs, EMBED and APPLET. Support for these (after a fashion) came in HTML 4.0
HTML 4.01 is the current official standard. It includes support for most of the proprietary extensions, plus support for extra features (Internationalized documents, support for Cascading Style Sheets, extra TABLE, FORM, and JavaScript enhancements), that are not universally supported. Some of these enhancements are not discussed in these online notes. They are discussed in detail in The HTML 4.0 Sourcebook, available in most bookstores.
HTML 5 Following its immediate predecessors HTML 4.01 and XHTML 1.1, HTML5 is a response to the observation that the HTML and XHTML in common use on the World Wide Web are a mixture of features introduced by various specifications, along with those introduced by software products such as web browsers, those established by common practice, and the many syntax errors in existing web documents. It is also an attempt to define a single markup language that can be written in either HTML or XHTML syntax.
